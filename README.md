# DevOps Production Labs 🚀

This repository documents my hands-on journey of becoming a production-ready DevOps engineer.

Instead of only studying theory, this project focuses on solving **real-world production scenarios** involving Linux servers and AWS infrastructure.

Each lab simulates a real incident that DevOps/SRE engineers commonly face in production environments.

The goal is to **investigate, debug, resolve, and document these problems using industry practices.**

---

# What This Repository Contains

This repository includes a collection of **daily DevOps labs** where I simulate infrastructure issues, troubleshoot them, and document the investigation process.

Each lab follows a structured engineering workflow:

Production Scenario  
↓  
Investigation  
↓  
Debugging  
↓  
Root Cause Analysis  
↓  
Resolution  
↓  
Verification  
↓  
Documentation

# Example Incident Scenarios

Some of the problems simulated in this repository include:

- Unable to SSH into EC2 instance
- Web server not responding
- Disk full causing application failure
- High CPU usage on Linux server
- Misconfigured security groups
- Network connectivity issues
- Service crashes

These are **common incidents that DevOps engineers troubleshoot in production environments.**

# Why This Project Exists

Modern DevOps roles require more than just knowing commands or cloud services.

Engineers must be able to:

- Understand system behavior
- Investigate failures
- Debug infrastructure issues
- Read logs and identify root causes
- Restore services quickly

This repository is my way of **building those real-world troubleshooting skills.**

---

# Learning Approach

Every lab follows a simple rule:

> "Break things. Understand why they break. Fix them."

This approach builds strong intuition around how systems behave in production.
